## 주요 의사결정

### 1. `@theme inline` → `@theme` 변경
- **문제**: `@theme inline`은 Tailwind가 CSS 변수 대신 값을 직접 인라인하여 `:root.dark` 오버라이드가 Tailwind 유틸리티 클래스에 반영되지 않음
- **해결**: `inline` 제거로 `bg-card-bg` 등이 `var(--color-card-bg)` 형태로 컴파일되어 다크모드 정상 작동

### 2. 탭바 배경색 분리
- **문제**: `bg-background`는 페이지 배경과 동일하여 탭바가 테마 전환 시 변화가 보이지 않음
- **해결**: `bg-card-bg` 적용 (라이트: #FFFFFF, 다크: #363636)

### 3. orange → accent 리네이밍
- **이유**: "orange"는 실제 색상과 무관 (라이트=민트, 다크=노란계열). 의미 기반 토큰명 `accent`로 통일
- **범위**: globals.css + designSystem.ts + 10개 컴포넌트/페이지 파일

### 4. 채팅 입력 UX 분리 (disabled vs sendDisabled)
- **기존**: AI 응답 중 textarea 자체가 `disabled` → 입력 불가
- **변경**: `sendDisabled` prop 분리 → AI 응답 중에도 타이핑 가능, 전송 버튼만 비활성화

### 5. 결과 분석 조기 요청
- **위치**: 채팅 입력창 바로 위 좌하단
- **활성 조건**: 4턴 이상 대화 진행
- **비활성 색상**: `var(--color-surface)` (회색)
- **활성 색상**: `var(--color-accent)` (라이트=민트 #A8D8C8, 다크=노랑 계열)
