# 008 — 맞춤 학습 설정 리디자인 + 디자인 시스템 구축

## 작업 요약

기존 "온보딩" 화면을 레퍼런스 이미지의 따뜻한 무드(베이지/머스타드 톤)로 전면 리디자인하고,
재사용 가능한 디자인 시스템 파일을 만들었습니다.

---

## 주요 변경 사항

### 1. 용어 및 라우트 변경

| 변경 전 | 변경 후 |
|---------|---------|
| 온보딩 | **맞춤 학습 설정** |
| `/onboarding` | **`/setup`** |

- 목적(사용자 기본정보 수집 + 수준 파악)에 맞는 용어로 변경
- 기존 `app/onboarding/`, `components/onboarding/` 폴더 삭제

### 2. 웰컴 화면 신규 생성

**파일**: `components/setup/WelcomeScreen.tsx`

- `/setup` 진입 시 첫 화면으로 표시
- 앱 로고 + 소개 문구 + 3가지 특징 + "맞춤 학습 설정 시작하기" CTA
- "약 1분이면 완료됩니다" 안내로 진입 장벽 낮춤

### 3. UI 전면 리디자인 (따뜻한 무드)

- **배경**: `#F5F0E8` 베이지 (라이트 톤 고정, 테마 토글 제거)
- **액센트**: `#D4A843` 머스타드 (기존 `#EF9F27` 주황 대비 부드러운 톤)
- **카드**: 흰색 배경 + `#E8E0D4` 보더 + 부드러운 그림자
- **진행 표시**: `1/4` 텍스트 + 바 → **닷 인디케이터** (현재 스텝은 길쭉한 바)
- **아이콘**: 이모지 → **lucide-react** 라인 아이콘
- **단계별 부제**: 각 단계의 목적 설명 추가 (예: "맞춤 대화를 위해 알려주세요")

### 4. 디자인 시스템 구축

**파일**: `lib/designSystem.ts`

| 항목 | 설명 |
|------|------|
| `WARM_THEME` 객체 | 따뜻한 무드 CSS 변수 토큰 모음 |
| `APP_THEME` 객체 | 기본 앱 테마 CSS 변수 토큰 모음 |
| `warmCardStyle(isSelected)` | 선택/미선택 카드 인라인 스타일 반환 |
| `warmIconContainerStyle(isSelected)` | 선택/미선택 아이콘 컨테이너 스타일 반환 |
| `warmCtaStyle(isEnabled)` | 활성/비활성 CTA 버튼 스타일 반환 |
| `warmPageStyle` | 따뜻한 무드 페이지 래퍼 스타일 |
| `COMMON_CLASSES` | 공통 Tailwind 클래스명 상수 |

추후 다른 화면에서도 `WARM_THEME`, `warmCardStyle()` 등을 import해서 동일한 무드를 적용할 수 있습니다.

### 5. 패키지 변경

- `lucide-react` 추가 (아이콘 패키지)
- `ts-jest` 27 → 최신 버전 업그레이드 (Jest 30 호환)

### 6. TDD 테스트

| 테스트 파일 | 테스트 수 | 내용 |
|------------|----------|------|
| `__tests__/onboardingValidation.test.ts` | 19개 | 유효성 검사 함수 전체 검증 |
| `__tests__/designSystem.test.ts` | 6개 | 디자인 시스템 헬퍼 함수 검증 |

## 용어 참조

| 용어 | 설명 |
|------|------|
| **디자인 시스템** | 색상·간격·스타일을 한 곳에 정의해서 모든 화면에서 일관되게 재사용하는 구조 |
| **디자인 토큰** | 색상·폰트·간격 등의 디자인 값을 변수로 정의한 것 (CSS 변수와 매핑) |
| **닷 인디케이터** | 진행 상황을 점(●)으로 표시하는 UI 패턴. 현재 단계는 길쭉한 바로 강조 |
| **lucide-react** | React용 오픈소스 아이콘 라이브러리. 깔끔한 선 스타일 아이콘 제공 |
| **트리쉐이킹** | 사용하지 않는 코드를 자동으로 제거해서 번들 크기를 줄이는 빌드 최적화 기법 |
| **TDD** | Test-Driven Development. 테스트를 먼저 작성하고, 테스트를 통과하는 코드를 작성하는 방식 |
