## 주요 설계 결정

### 채팅 UI (F-03)
- **스트리밍 효과**: `StreamingBubble` 컴포넌트에서 `setInterval`로 글자 하나씩 표시 (속도: 40ms/글자)
- **타이핑 인디케이터**: AI 응답 대기 중 점 3개 바운스 애니메이션
- **입력창**: `textarea` + 높이 자동 조절 (최대 4줄, 96px), Enter 전송 / Shift+Enter 줄바꿈
- **자동 스크롤**: 메시지 추가/스트리밍 시 하단으로 smooth scroll
- **대화 종료**: `isFinished` 감지 후 1.5초 딜레이 → `/result` 이동

### 결과 화면 (F-04)
- **총점**: 5자리 큰 숫자 + 10개 별점 시각화
- **티어**: 티어별 HEX 색상으로 카드 테두리 + 아이콘 색상 동적 적용
- **3축 점수**: SVG 원형 게이지 (circumference 기반 진행도)
- **레이더 그래프**: recharts + dynamic import (SSR 비호환 우회)

### 피드백 UI (F-05a)
- **오답 하이라이트**: 사용자 발화 말풍선 내 오답 부분에 빨간 배경 + wavy underline
- **오답 말풍선**: 정상 메시지(주황 배경)와 오답 메시지(연한 빨강 배경+빨강 테두리) 시각 구분
- **교정 카드**: 원래 표현(취소선) → 올바른 표현(초록) + 설명
- **대화 로그**: 기본 4건 표시, "전체 보기" 토글로 펼침

### 방사형 그래프 (F-05b)
- **라이브러리**: recharts 선택 (React 생태계 표준, 번들 크기 적절)
- **SSR 이슈**: `react-is` 모듈 SSR 미지원 → `next/dynamic`으로 클라이언트 전용 로드
- **3축**: 어휘(30%) / 상황(50%) / 문법(20%), 0~10 스케일

## 기술 용어 참조

- **스트리밍(Streaming)**: 데이터를 한꺼번에 받지 않고 조금씩 받아서 표시하는 방식. 여기서는 AI 답변 글자를 하나씩 보여줌
- **dynamic import**: 페이지 로드 시 바로 불러오지 않고, 필요할 때 불러오는 방식. SSR(서버에서 HTML 만들기)과 호환되지 않는 라이브러리에 사용
- **recharts**: React용 차트 라이브러리. 막대/선/레이더 등 다양한 차트를 그릴 수 있음
- **circumference**: 원의 둘레. SVG 원형 게이지에서 진행도를 표현할 때 사용 (총 둘레 중 몇 % 채울지 계산)
- **wavy underline**: 물결 모양 밑줄. 오타나 오답을 표시할 때 자주 사용되는 시각적 표현
