### TODO 22 — 스포트라이트 오버레이 컴포넌트
**파일**: `platform/components/tutorial/TutorialOverlay.tsx`

어두운 반투명 배경에서 특정 요소만 밝게 보이는 효과를 구현했습니다.

**구멍 뚫기 원리**: CSS `clip-path`의 `polygon`으로 전체 화면을 덮는 도형에서 타겟 사각형 영역을 반대 방향으로 잘라냄. `evenodd` fill-rule에서 겹치는 영역이 투명해지는 성질을 이용.

**타겟 위치 측정**: `document.getElementById(targetId).getBoundingClientRect()`로 DOM 요소의 실제 화면 좌표를 읽어옴. 화면 리사이즈 시 재측정.

기능 목록:
- 타겟 요소 주위 주황 테두리 강조
- 상단 좌: `N / 8` 진행 표시
- 상단 우: "건너뛰기" 버튼 (항상 노출)
- 말풍선 위치: 타겟 위(`top`) 또는 아래(`bottom`) 선택 가능
- 화면 탭 → 다음 스텝
- 마지막 스텝: "준비됐어요! 첫 대화를 시작해볼까요? 🎉" 완료 버튼 추가

## 기술 용어 주석

- **`clip-path: polygon(...)`**: CSS로 요소의 보이는 영역을 다각형으로 잘라내는 속성. 좌표 목록을 나열해 원하는 모양만 보이게 할 수 있음.
- **`getBoundingClientRect()`**: DOM 요소가 화면(viewport)에서 차지하는 위치와 크기를 반환하는 메서드. `top`, `left`, `width`, `height` 값을 픽셀로 제공.
- **`pointer-events-none`**: Tailwind 클래스. 이 요소는 마우스/터치 이벤트를 받지 않음 (클릭이 통과됨). 오버레이 마커 div에 사용해 실제 버튼 클릭을 막지 않게 함.
- **`setTimeout(..., 300)`**: 300밀리초(0.3초) 후에 함수를 실행. 페이지가 완전히 그려진 뒤 DOM 요소 위치를 측정하기 위한 지연.
- **`useCallback`**: React 훅. 함수를 메모이제이션(캐싱)하여, 불필요한 재생성을 방지. `useEffect`의 의존성 배열에 함수를 넣을 때 유용.
- **`router.replace()` vs `router.push()`**: push는 히스토리에 추가(뒤로가기 가능), replace는 현재 항목을 교체(뒤로가기 불가). 온보딩 리디렉트에 replace 사용.
